docker ps

A quick guide on using docker ps to view running containers and troubleshooting common Postgres container execution issues.

Example: docker ps

docker ps
CONTAINER ID   IMAGE                            COMMAND                  CREATED      STATUS                 PORTS                                                           NAMES
b7b51543e151   quay.io/keycloak/keycloak:26.0   "/opt/keycloak/bin/k…"   4 days ago   Up 2 hours             8443/tcp, 0.0.0.0:8080->8080/tcp, :::8080->8080/tcp, 9000/tcp   ticket_keycloak
e5311074829b   postgres:17-alpine               "docker-entrypoint.s…"   4 days ago   Up 2 hours (healthy)   0.0.0.0:5434->5432/tcp, [::]:5434->5432/tcp                     ticket-generator-postgres

Troubleshooting Exec

rinas@HMS-LP-163:~/Desktop/project/ticket-generator$ docker exec e5311074829b -it
OCI runtime exec failed: exec failed: unable to start container process: exec: "-it": executable file not found in $PATH: unknown

Connecting to Postgres

rinas@HMS-LP-163:~/Desktop/project/ticket-generator$ docker inspect e5311074829b | grep POSTGRES
                "POSTGRES_USER=dev_user",
                "POSTGRES_PASSWORD=dev_password",
                "POSTGRES_HOST_AUTH_METHOD=trust",
                "POSTGRES_DB=ticket_generator_dev",
rinas@HMS-LP-163:~/Desktop/project/ticket-generator$ psql -h localhost -p 5434 -U dev_user -d ticket_generator_dev
